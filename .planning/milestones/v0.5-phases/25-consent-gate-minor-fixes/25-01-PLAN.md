    **Step 1: Navigation types and route registration**

    **Step 2: PIN service**

    Create `src/services/consent/parentalPin.ts` with three exported async functions:
    - `hasParentalPin()`: checks SecureStore for key `'parental-pin'`, returns boolean
    - `setParentalPin(pin: string)`: stores via `SecureStore.setItemAsync('parental-pin', pin)`
    - `verifyParentalPin(input: string)`: retrieves stored PIN, compares with input, returns boolean

    Use the same import pattern as geminiClient.ts: `import * as SecureStore from 'expo-secure-store'`.

    **Step 3: ConsentScreen**

    Create `src/screens/ConsentScreen.tsx` as a single screen with two modes determined on mount:
    - On mount, call `hasParentalPin()` to determine `'create'` vs `'verify'` mode (with a `'loading'` initial state)
    - Use `useState` for: mode ('loading'|'create'|'verify'), enteredPin (string), confirmPin (string), pinStep ('enter'|'confirm' for create mode), error message

    **Layout (single scrollable screen):**
    1. Header: Shield icon (from lucide-react-native) + "AI Helper Setup" title
    2. Info section: Warm paragraph + 3-4 bullet points about safeguards:
       - "No personal information is shared with the AI"
       - "All responses are age-appropriate and checked"
       - "Your child uses buttons only -- no typing"
       - "You can turn this off anytime"
    3. PIN section header: "Create a PIN" (create mode) or "Enter your PIN" (verify mode)
       - In create mode, show "Enter a 4-digit PIN" then after 4 digits switch to "Confirm your PIN"
    4. Four dot indicators: filled (colors.primary) when digit entered, unfilled (colors.surface)
    5. Number pad: 3x4 grid of circular buttons (1-9, empty, 0, backspace) using dark theme colors. Button size ~64px with colors.surface background, colors.textPrimary text. Min touch target 48dp per CLAUDE.md.
    6. Error message area below dots (colors.incorrect text)

    **PIN creation flow:**
    - User enters 4 digits -> enteredPin fills -> auto-advance to confirm step (clear dots, show "Confirm your PIN")
    - User enters 4 digits again -> if match: call `setParentalPin(enteredPin)`, then `setTutorConsentGranted(true)`, then `navigation.goBack()`
    - If mismatch: show "PINs don't match. Try again." error, reset both pins, go back to enter step

    **PIN verification flow:**
    - User enters 4 digits -> call `verifyParentalPin(enteredPin)`
    - If true: call `setTutorConsentGranted(true)`, then `navigation.goBack()`
    - If false: show "Wrong PIN. Try again." error, clear enteredPin and dot indicators

    CRITICAL: Call `setTutorConsentGranted(true)` BEFORE `navigation.goBack()` to avoid race condition (Zustand sync update guarantees store is updated before next render).

    Use `StyleSheet.create` for all styles. Import colors, spacing, typography, layout from `@/theme`. Use `useSafeAreaInsets` for safe area padding. Use `ScrollView` for the outer container to handle small screens.

    Import `Shield`, `Delete` (backspace icon) from `lucide-react-native`.
