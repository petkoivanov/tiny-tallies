First, add the new type to `sessionTypes.ts`:
```typescript
/** Category tag for practice problems in the 60/30/10 mix */
export type PracticeProblemCategory = 'review' | 'new' | 'challenge';

/** Slot allocation for the practice portion of a session */
export interface PracticeSlotCounts {
  readonly review: number;
  readonly new: number;
  readonly challenge: number;
}
```

Then create `src/services/session/practiceMix.ts` with these pure functions:

1. `computeSlotCounts(practiceCount: number): PracticeSlotCounts` -- Compute 60/30/10 split with rounding. Use: challenge = Math.round(count * 0.1), new = Math.round(count * 0.3), review = count - new - challenge. This ensures sum always equals practiceCount. review gets the remainder so it never drops below intended.

2. `buildReviewPool(skillStates, now?: Date): Array<{skillId: string, masteryPL: number, overdueByMs: number}>` -- Iterate all skill IDs in skillStates, call getReviewStatus on each, keep only isDue=true AND masteryLocked=false, sort by lowest P(L) first (weakest due skills prioritized).

3. `buildNewSkillPool(skillStates): Array<{skillId: string, masteryPL: number}>` -- Call getOuterFringe(skillStates), map each to include P(L) (default 0.1 for unpracticed).

4. `buildChallengePool(skillStates): Array<{skillId: string, masteryPL: number, eloRating: number}>` -- From getUnlockedSkills, filter to: masteryLocked=false AND P(L) in [0.40, 0.80] range AND attempts > 0 (must have been practiced). Include eloRating for template selection.

5. `selectFromPool<T extends {skillId: string, masteryPL: number}>(pool: T[], rng: SeededRng, usedSkillIds: Set<string>): T | null` -- BKT-weighted selection with inverse P(L) weighting. Weight = (1 - masteryPL) + 0.05 floor. First try selecting from pool items NOT in usedSkillIds. If all pool items are already used, select from full pool (allow repeats). Return null if pool is empty.

6. `generatePracticeMix(skillStates, childAge: number | null, rng: SeededRng, practiceCount?: number, now?: Date): Array<{skillId: string, category: PracticeProblemCategory}>` -- Main orchestrator function:
   - Compute slot counts
   - Build all three pools
   - For each category in order (review, new, challenge), select skills using selectFromPool
   - Apply fallback cascade:
     a. Unfilled challenge slots -> try review pool, then new pool
     b. Unfilled new slots -> try review pool
     c. Unfilled review slots -> try new pool
     d. If ALL pools empty -> use getUnlockedSkills with weakness-weighted selection (existing selectSkill)
     e. If truly nothing available (brand new user, no unlocked skills... impossible due to root skills, but safety) -> use root skills
   - Track usedSkillIds across all selections for unique-skill preference
   - Return array of {skillId, category} before ordering

7. `constrainedShuffle(items: Array<{skillId: string, category: PracticeProblemCategory}>, rng: SeededRng): Array<{skillId: string, category: PracticeProblemCategory}>` -- Constrained random ordering:
   - Start with one review problem (first slot guaranteed review)
   - Remaining items shuffled with constraint: no two challenge items adjacent
   - Algorithm: Fisher-Yates on remaining items, then swap any adjacent challenge pairs by inserting a non-challenge between them. If impossible (too many challenges), just do best-effort.
