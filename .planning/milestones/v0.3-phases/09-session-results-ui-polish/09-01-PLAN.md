**4. Polish SessionScreen.tsx UI:**

a) **Add progress bar below header:**
- Add a new View below the header row containing the progress bar
- Bar: full width with horizontal padding (spacing.lg), height 8, borderRadius round, background colors.surface
- Fill: borderRadius round, height 100%, width = `${((currentIndex + (feedbackState ? 1 : 0)) / totalProblems) * 100}%`
  - When feedback is showing (answer submitted), count that question as "done" in the bar
- Color-code fill by session phase using these colors (Claude's discretion):
  - Warmup: colors.primaryLight (#818cf8, soft indigo)
  - Practice: colors.primary (#6366f1, full indigo)
  - Cooldown: colors.correct (#84cc16, lime green — session wind-down feels accomplished)
- Keep the text count and phase label in the header row above

b) **Remove the separate feedback icon/text container:**
- Delete the entire `{feedbackState && ( ... )}` block (lines 145-173) that renders the Check/X icon and "Correct!"/"Not quite" text
- Remove the `Check` and `X` imports from lucide-react-native (the quit button X is still needed — keep that X import)
- Actually, `X` is used by the quit button. Only remove `Check` from lucide imports.

c) **Add answer button feedback coloring:**
- When `feedbackState` is active, color the buttons based on selection:
  - The button matching `selectedAnswer` gets:
    - If correct: green border (2px, colors.correct) + slight green-tinted background (`#84cc1620`)
    - If incorrect: red border (2px, colors.incorrect) + slight red-tinted background (`#f8717120`)
  - After the ~500ms delay, the correct answer button gets green border + background
    - Implementation: Add a `showCorrectAnswer` boolean state. Set it to true via a separate 500ms setTimeout inside handleAnswer (only when answer is wrong). Reset it alongside feedbackState when advancing.
  - Buttons that are neither selected nor the correct reveal stay at their default surface color with reduced opacity (current disabled style)

d) **Add scale-on-press effect:**
- Use React Native's `Animated` API (not Reanimated — simpler for a single scale transform, Reanimated is heavier setup).
- Create an `Animated.Value` for scale per button, or simpler: use the `Pressable` style callback with `pressed` state and apply `transform: [{ scale: 0.95 }]` when pressed.
- Since Pressable already supports `({ pressed }) => [...]` style callbacks, add `pressed && { transform: [{ scale: 0.95 }] }` to each option button's style.
- This gives instant visual response without needing Animated API at all.

e) **Style cleanup:**
- Remove `feedbackContainer` and `feedbackText` styles (no longer used)
- Add new styles: `progressBarContainer`, `progressBarBackground`, `progressBarFill`, `optionButtonCorrect`, `optionButtonIncorrect`, `optionButtonRevealCorrect`
