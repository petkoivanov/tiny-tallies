  <action>
Create `src/services/gamification/levelProgression.ts` with these pure functions:

1. `calculateXpForLevel(level: number): number` — Returns cumulative XP threshold to REACH a given level. Level 1 requires 0 XP (starting level). Level 2 requires 120 XP (100 + 1*20). Level 3 requires 120 + 140 = 260 XP (100 + 2*20). This is a cumulative sum: sum from i=1 to level-1 of (100 + i*20). Use a loop or closed-form arithmetic.

2. `calculateLevelFromXp(totalXp: number): { level: number; xpIntoCurrentLevel: number; xpNeededForNextLevel: number }` — Given cumulative total XP, derives current level, how much XP is "into" the current level (for progress bar), and how much XP the next level needs. Walk levels upward until totalXp < threshold for next level. Level starts at 1.

3. `detectLevelUp(previousTotalXp: number, newTotalXp: number): { leveledUp: boolean; previousLevel: number; newLevel: number; levelsGained: number }` — Compares old vs new total XP to detect level changes. Uses calculateLevelFromXp internally. Supports multi-level jumps (levelsGained can be > 1).

Export constants:
- `BASE_LEVEL_XP = 100` (base XP per level)
- `LEVEL_XP_INCREMENT = 20` (additional XP per level)

Create `src/services/gamification/index.ts` barrel export for all functions and constants.

Create `src/__tests__/gamification/levelProgression.test.ts` with tests:
- calculateXpForLevel: level 1 = 0, level 2 = 120, level 3 = 260, level 4 = 420
- calculateLevelFromXp: 0 XP = level 1 (0/120), 119 XP = level 1 (119/120), 120 XP = level 2 (0/140), 250 XP = level 2 (130/140), 260 XP = level 3
- detectLevelUp: no level change (0 -> 50), single level up (50 -> 130), multi-level jump (0 -> 300), no change when XP same
- Edge case: very large XP values work correctly (e.g., 10000 XP)
