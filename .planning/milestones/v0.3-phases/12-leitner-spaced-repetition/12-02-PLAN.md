## Goal

Wire Leitner box transitions into the session answer flow and commit path so that every answered problem updates the skill's Leitner box, nextReviewDue, and graduation status. After this plan, each skill's spaced repetition state is updated atomically alongside Elo and BKT on session completion.

## Context

### Locked Decisions (from 12-CONTEXT.md — NON-NEGOTIABLE)
- BKT is the sole authority for mastery declarations — Leitner boxes control review TIMING only
- Box 6 graduation is a scheduling milestone, not a mastery declaration
- Correct -> move up 1 box (max Box 6)
- Wrong -> drop 2 boxes (min Box 1)
- Wall-clock timestamps for nextReviewDue
- Auto-advance box to 6 when BKT declares mastery (Claude's discretion — chosen for consistency)

### What 12-01 Provides
- `transitionBox(currentBox, isCorrect, consecutiveCorrectInBox6, childAge, now)` returns `LeitnerTransitionResult`
- `SkillState` extended with `leitnerBox`, `nextReviewDue`, `consecutiveCorrectInBox6`
- `PendingSkillUpdate` extended with `newLeitnerBox`, `newNextReviewDue`, `newConsecutiveCorrectInBox6`
- `getReviewStatus()` for due checks
- `computeNextReviewDue()` for scheduling

### Integration Points
- `useSession.ts` handleAnswer callback (lines ~142-210): accumulates pending updates per skill. Must add Leitner transition alongside existing Elo + BKT updates.
- `sessionOrchestrator.ts` commitSessionResults (lines ~179-230): iterates pendingUpdates and calls updateSkillState. Must include Leitner fields in the commit.

## must_haves

Derived goal-backward from phase success criteria:

1. **handleAnswer computes Leitner transition for every answered problem** — Without this, box positions never update during practice (LEIT-01, LEIT-02, LEIT-03)
2. **Leitner transition uses childAge for age-adjusted intervals** — Younger children get shorter review gaps in computed nextReviewDue (LEIT-04)
3. **commitSessionResults persists leitnerBox, nextReviewDue, consecutiveCorrectInBox6 to store** — Without atomic commit, Leitner state is lost on session end (LEIT-01)
4. **BKT mastery auto-advances box to 6** — Keeps Leitner scheduling consistent with BKT mastery declarations
5. **consecutiveCorrectInBox6 flows through pending updates to commit** — Graduation tracking must persist across sessions (LEIT-05)
6. **All existing tests pass with no regressions** — Leitner is additive, must not break Elo/BKT/XP flows
