## Goal

Create the Leitner spaced repetition service as a pure-function module following the BKT calculator pattern, extend the SkillState type with Leitner fields, and add store migration v3->v4. This plan delivers the complete data model and computation engine; the next plan wires it into the session flow.

## Context

### Locked Decisions (from 12-CONTEXT.md — NON-NEGOTIABLE)
- BKT is the sole authority for mastery declarations — Leitner boxes control review TIMING only
- Box 6 graduation (3 consecutive correct in Box 6) is a scheduling milestone, not a mastery declaration
- Correct -> move up 1 box (max Box 6)
- Wrong -> drop 2 boxes (min Box 1), NOT all the way to Box 1
- Mastered (Box 6 graduated) -> monthly review schedule
- Age-adjusted intervals: shorter gaps for younger children (discrete brackets: 6-7, 7-8, 8-9)
- Wall-clock timestamps: store nextReviewDue as ISO date string, compare against current date
- BKT-informed initial placement: use existing P(L) to place skills in appropriate boxes during migration
- Skills with no BKT data start in Box 1

### Existing Code to Follow
- `bktCalculator.ts` pattern: pure functions, exported constants, age bracket lookup
- `bktTypes.ts` pattern: separate types file
- `skillStatesSlice.ts`: SkillState type — extend, do not replace
- `skillStateHelpers.ts`: `getOrCreateSkillState` default object — must add Leitner defaults
- `migrations.ts`: chain pattern with `if (version < N)` blocks
- `appStore.ts`: STORE_VERSION constant, partialize config
- `sessionTypes.ts`: PendingSkillUpdate — must add Leitner fields

### Age-Adjusted Interval Table (from .planning/06-spaced-repetition.md)
| Box | Age 6-7      | Age 7-8  | Age 8-9  |
|-----|-------------|----------|----------|
| 1   | Same session | Same session | Same session |
| 2   | Same day (2hr) | Next day | Next day |
| 3   | 1 day       | 2 days   | 3 days   |
| 4   | 3 days      | 5 days   | 7 days   |
| 5   | 7 days      | 10 days  | 14 days  |
| 6   | 14 days     | 21 days  | 30 days  |
| Mastered (graduated) | 30 days | 30 days | 30 days |

## Tasks

<task id="12-01-1" title="Create Leitner types and calculator service">
<description>
Create `src/services/adaptive/leitnerTypes.ts` and `src/services/adaptive/leitnerCalculator.ts` following the exact pattern of bktTypes.ts / bktCalculator.ts.

**leitnerTypes.ts** — Define:
- `LeitnerBox` type: `1 | 2 | 3 | 4 | 5 | 6`
- `LeitnerTransitionResult` interface: `{ newBox: LeitnerBox; nextReviewDue: string; consecutiveCorrectInBox6: number; graduated: boolean }`
- `LeitnerReviewStatus` interface: `{ isDue: boolean; overdueByMs: number }` (for Phase 14 queue sorting)

**leitnerCalculator.ts** — Implement pure functions:

1. `LEITNER_INTERVALS` — Exported constant: Record mapping age bracket string ("6-7", "7-8", "8-9") to an array of 6 interval values in **milliseconds** (one per box). Use the research table above. Box 1 = 0ms (same session). Box 2 for age 6-7 = 2 * 60 * 60 * 1000 (2 hours). Other boxes: multiply days * 24 * 60 * 60 * 1000. Also include a `"default"` key that maps to "7-8" values.

2. `GRADUATED_REVIEW_INTERVAL_MS` — Exported constant: 30 days in ms (30 * 24 * 60 * 60 * 1000).

3. `getAgeIntervalBracket(childAge: number | null): string` — Maps childAge to bracket key: 6 or 7 -> "6-7", 8 -> "7-8", 9 -> "8-9", null -> "default".

4. `getIntervalMs(box: LeitnerBox, childAge: number | null): number` — Returns the interval in ms for a given box and age. Looks up bracket from `getAgeIntervalBracket`, then indexes into `LEITNER_INTERVALS[bracket][box - 1]`.

5. `computeNextReviewDue(box: LeitnerBox, childAge: number | null, now?: Date): string` — Returns ISO string. Calls `getIntervalMs` and adds to `now.getTime()`, returns `new Date(sum).toISOString()`.

6. `transitionBox(currentBox: LeitnerBox, isCorrect: boolean, consecutiveCorrectInBox6: number, childAge: number | null, now?: Date): LeitnerTransitionResult` — Core box transition logic:
   - If correct: newBox = min(currentBox + 1, 6) as LeitnerBox
   - If wrong: newBox = max(currentBox - 2, 1) as LeitnerBox
   - Track consecutiveCorrectInBox6: if correct AND currentBox === 6, increment counter; if wrong, reset to 0; if correct but currentBox !== 6, reset to 0
   - `graduated` = (newBox === 6 && updated consecutiveCorrectInBox6 >= 3)
   - `nextReviewDue`: if graduated, use `GRADUATED_REVIEW_INTERVAL_MS`; else use `computeNextReviewDue(newBox, childAge, now)`
   - Default `now` to `new Date()` if not provided

7. `getReviewStatus(nextReviewDue: string | null, now?: Date): LeitnerReviewStatus` — Returns { isDue, overdueByMs }. If nextReviewDue is null, return { isDue: true, overdueByMs: 0 } (new skills are immediately due). Parse nextReviewDue as Date, compare with now. isDue = now >= dueDate. overdueByMs = max(0, now - dueDate).

8. `mapPLToInitialBox(masteryProbability: number): LeitnerBox` — BKT-informed initial placement for migration:
   - P(L) < 0.20 -> Box 1
   - P(L) < 0.40 -> Box 2
   - P(L) < 0.60 -> Box 3
   - P(L) < 0.80 -> Box 4
   - P(L) < 0.95 -> Box 5
   - P(L) >= 0.95 -> Box 6

**Barrel export:** Update `src/services/adaptive/index.ts` to export all types and functions from both new files.
</description>
</task>

```typescript
if (version < 4) {
  // v3 -> v4: Add Leitner spaced repetition fields to existing skill states
  const skillStates = (state.skillStates ?? {}) as Record<string, Record<string, unknown>>;
  for (const skillId of Object.keys(skillStates)) {
    const skill = skillStates[skillId];
    // BKT-informed initial box placement
    const masteryProbability = (skill.masteryProbability as number) ?? 0.1;
    skill.leitnerBox ??= mapPLToInitialBox(masteryProbability);
    skill.nextReviewDue ??= null; // null = immediately due for review
    skill.consecutiveCorrectInBox6 ??= 0;
  }
  state.skillStates = skillStates;
}
```
Import `mapPLToInitialBox` from `../../services/adaptive/leitnerCalculator` at the top of migrations.ts.
