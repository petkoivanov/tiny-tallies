  <behavior>
    ## MisconceptionRecord type
    - Has fields: bugTag (string), skillId (string), occurrenceCount (number), status ('new' | 'suspected' | 'confirmed'), firstSeen (ISO string), lastSeen (ISO string)

    ## MisconceptionSlice state
    - `misconceptions`: Record keyed by composite `${bugTag}::${skillId}` string -> MisconceptionRecord
    - `sessionRecordedKeys`: Set/array of composite keys already recorded this session (for deduplication)

    ## Actions
    - `recordMisconception(bugTag: string, skillId: string)`:
      - Builds composite key `${bugTag}::${skillId}`
      - If key is in sessionRecordedKeys, returns early (dedup)
      - If no existing record: creates new MisconceptionRecord with occurrenceCount=1, status='new', firstSeen=now, lastSeen=now
      - If existing record: increments occurrenceCount, updates lastSeen
      - Adds key to sessionRecordedKeys
    - `resetSessionDedup()`: Clears sessionRecordedKeys (called on session start)
    - `getMisconceptionsBySkill(skillId)` — selector: returns all records for a given skillId
    - `getMisconceptionsByBugTag(bugTag)` — selector: returns all records for a given bugTag

    ## Test cases
    - Initial state: empty misconceptions map, empty sessionRecordedKeys
    - recordMisconception creates a new record with correct fields
    - recordMisconception increments existing record's occurrenceCount and updates lastSeen
    - recordMisconception with same bugTag+skillId in same session is deduplicated (count stays at 1)
    - recordMisconception after resetSessionDedup allows recording same key again (new session)
    - Same bugTag with different skillId creates separate records
    - getMisconceptionsBySkill returns only records matching skillId
    - getMisconceptionsByBugTag returns only records matching bugTag
    - getMisconceptionsBySkill/ByBugTag return empty array when no matches

    ## Migration test cases
    - v6->v7 migration initializes `misconceptions` as empty object `{}`
    - v6->v7 migration preserves all existing v6 state (childName, skillStates, etc.)
    - Full chain migration v1->v7 adds misconceptions field alongside all prior fields
  </behavior>
