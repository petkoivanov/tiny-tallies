Refactor `generateSessionQueue` in `sessionOrchestrator.ts`:

1. **Update signature** -- Add `childAge: number | null = null` parameter after `config`:
   ```typescript
   export function generateSessionQueue(
     skillStates: Record<string, SkillState>,
     config: SessionConfig = DEFAULT_SESSION_CONFIG,
     childAge: number | null = null,
     seed: number = Date.now(),
   ): SessionProblem[]
   ```
   Note: `childAge` is inserted before `seed` because `seed` has a default. This BREAKS the old call signature where seed was the 3rd arg. All existing callers pass seed as 3rd positional arg, so we need to update them. However, since the only caller patterns in tests are `generateSessionQueue({}, DEFAULT_SESSION_CONFIG, 42)`, the 3rd arg (42) would now become childAge. To maintain backward compatibility and avoid a breaking change, instead add childAge as the LAST parameter:
   ```typescript
   export function generateSessionQueue(
     skillStates: Record<string, SkillState>,
     config: SessionConfig = DEFAULT_SESSION_CONFIG,
     seed: number = Date.now(),
     childAge: number | null = null,
   ): SessionProblem[]
   ```
   This preserves the existing 3-arg call pattern.

2. **Replace practice loop** -- Instead of the current for-loop that uses `selectSkill` for practice, do:
   ```typescript
   // Generate practice mix
   const practiceMix = generatePracticeMix(skillStates, childAge, rng, config.practiceCount);
   const orderedMix = constrainedShuffle(practiceMix, rng);
   ```
   Then for each practice slot, use the category to determine template selection:
   - `review` and `new` categories: Use `selectTemplateForSkill(skillId, skillState.eloRating, rng)` -- standard gaussian-targeted selection (same as before)
   - `challenge` category: Use `selectChallengeTemplate(skillId, skillState.eloRating, rng)` -- a new helper that filters templates to those with baseElo > childElo, then uses weightedRandomSelect. If no templates above Elo, fall back to standard selectTemplateForSkill.

3. **Add `selectChallengeTemplate` helper** (not exported, internal to sessionOrchestrator.ts):
   ```typescript
   function selectChallengeTemplate(skillId: string, studentElo: number, rng: SeededRng): ProblemTemplate {
     const templates = getTemplatesBySkill(skillId);
     const harder = templates.filter(t => t.baseElo > studentElo);
     if (harder.length === 0) {
       // No harder templates -- fall back to standard selection
       return selectTemplateForSkill(skillId, studentElo, rng);
     }
     // Use gaussian weighting centered above student Elo
     const weighted = weightBySuccessProbability(studentElo, harder);
     return weightedRandomSelect(weighted, rng);
   }
   ```

4. **Import additions** -- Add imports for `generatePracticeMix`, `constrainedShuffle` from `./practiceMix`, and `weightBySuccessProbability`, `weightedRandomSelect` from adaptive barrel.

5. **Update barrel** in `src/services/session/index.ts` -- No new exports needed (generateSessionQueue signature change is backward compatible). Ensure `PracticeProblemCategory` and `PracticeSlotCounts` are already exported from Plan 01.

Keep `commitSessionResults`, `getSessionPhase`, `selectStrongestSkill`, `selectEasiestTemplate` completely unchanged.
