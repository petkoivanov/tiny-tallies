<objective>
Implement the Bug Library -- a catalog of misconception-based distractor compute functions for addition and subtraction, plus a three-phase assembly algorithm that produces exactly 3 distractors per problem.

Purpose: Problems need plausible wrong answers that reflect real student misconceptions (e.g., forgetting to carry, subtracting smaller from larger per column). This enables diagnostic value in Phase v0.5 misconception detection and makes multiple choice answers educationally meaningful rather than random.

Output: `src/services/mathEngine/bugLibrary/` module with types, 11 bug patterns (6 addition, 5 subtraction), validation utilities, and the `generateDistractors()` assembly function.
</objective>

    ## Addition Bugs (additionBugs.ts) -- 6 patterns
    - add_no_carry: 27+18 -> 35 (ignores carry, each column independent). Returns null when no carry required.
    - add_carry_wrong_column: 27+18 -> 315 (writes full column sum instead of carrying). Returns null when no carry or when result equals correct answer.
    - add_off_by_one_plus: correct+1. Returns null when would equal correct answer (impossible for +1 unless overflow).
    - add_off_by_one_minus: correct-1. Returns null when correctAnswer <= 1.
    - add_concat: 3+4 -> 34. Only for single-digit operands (both a and b < 10). Returns null for multi-digit.
    - add_reverse_digits: 9+4=13 -> 31. Returns null when correctAnswer < 10 or tens digit equals ones digit.

    ## Subtraction Bugs (subtractionBugs.ts) -- 5 patterns
    - sub_smaller_from_larger: 42-17 -> 35 (subtracts smaller digit from larger per column). Returns null when no borrow required.
    - sub_no_borrow: 53-28 -> 35 (ones: 3<8 so writes |3-8|=5, tens: 5-2=3; mimics smaller-from-larger without carrying). Returns null when no borrow required.
    - sub_borrow_forget_reduce: 53-28 -> 35 (borrows for ones giving 13-8=5, but does not reduce tens: 5-2=3 instead of 4-2=2). Returns null when no borrow required.
    - sub_off_by_one_plus: correct+1. Returns null when would equal operand a (plausibility).
    - sub_off_by_one_minus: correct-1. Returns null when correctAnswer <= 0.

    ## generateDistractors(problem, rng) -> DistractorResult[]
    - Always returns exactly 3 DistractorResult items
    - No distractor value equals correctAnswer
    - All distractor values are unique (no duplicates among the 3 + the correctAnswer)
    - All distractors pass isValidDistractor()
    - Phase 1 bug-library distractors are attempted first (target up to 2 from bug patterns)
    - Phase 2 off-by-one adjacent distractor fills next slot
    - Phase 3 random fallback fills remaining slots
    - Deterministic given same problem + seed

    ## Edge Cases
    - Simple problem 1+1=2: few bugs apply, must still produce 3 distractors via fallback
    - Large problem 100+200=300: bug patterns like no-carry produce correct answer (filtered), fallback covers
    - Subtraction 10-9=1: tight range, must produce 3 distractors > 0
    - Problem where all bugs return null: relies entirely on adjacent + random phases

    ## Barrel Export (index.ts)
    - Re-exports: generateDistractors, DistractorResult, DistractorSource, BugPattern, isValidDistractor, shuffleArray, ADDITION_BUGS, SUBTRACTION_BUGS

    Create `src/services/mathEngine/bugLibrary/distractorGenerator.ts`:
    - Import Problem and Operation from `../types`, SeededRng from `../seededRng`, BugPattern and DistractorResult from `./types`, ADDITION_BUGS from `./additionBugs`, SUBTRACTION_BUGS from `./subtractionBugs`, isValidDistractor and shuffleArray from `./validation`

    - Helper: `getApplicableBugs(problem: Problem): BugPattern[]`
      - Select bugs by operation (ADDITION_BUGS for addition, SUBTRACTION_BUGS for subtraction)
      - Filter by minDigits <= problem.metadata.digitCount
      - Exclude off-by-one patterns (they go in Phase 2 of the algorithm, not Phase 1)
      - Return filtered array

    - Main function: `generateDistractors(problem: Problem, rng: SeededRng): DistractorResult[]`
      - Initialize `results: DistractorResult[] = []` and `used = new Set<number>([problem.correctAnswer])`
      - **Phase 1 -- Bug Library (target 2):**
        - Get applicable bugs, shuffle them with rng for variety
        - For each bug, compute `bug.compute(problem.operands[0], problem.operands[1], problem.operation)`
        - If result is not null, not in `used`, and passes `isValidDistractor()`: add to results, add to used
        - Stop after 2 bug-library distractors
      - **Phase 2 -- Off-by-one adjacent (target 1):**
        - If results.length < 3: try correctAnswer + 1 (if valid and not in used), else try correctAnswer - 1
        - Add with source: 'adjacent'
      - **Phase 3 -- Random fallback:**
        - While results.length < 3:
          - Compute a random value in a plausible range: `[Math.max(1, correctAnswer - Math.max(Math.floor(correctAnswer * 0.4), 5)), correctAnswer + Math.max(Math.floor(correctAnswer * 0.4), 5)]`
          - Use rng.intRange() to pick the value
          - If not in used and passes isValidDistractor(): add with source: 'random'
          - Safety: limit loop iterations to 50 to prevent infinite loops on extremely tight ranges. If 50 iterations hit, use deterministic offset: try correctAnswer + 2, +3, +4, etc. until 3 distractors found
        - Return results (guaranteed length === 3)
