Create the NumberLine manipulative with SVG-based rendering and animated marker (~300 lines main + ~150 lines SVG + ~30 lines types).

**NumberLineTypes.ts (~30 lines):**
- `interface NumberLineProps { range?: [number, number]; startPosition?: number; testID?: string; }`
- Default range: `[0, 10]`
- `interface HopArrow { fromValue: number; toValue: number; }` — represents one hop arc
- `const NUMBER_LINE_COLOR = '#4ADE80'` (green per CONTEXT.md)
- `const NUMBER_LINE_ACCENT = '#22C55E'`
- `const TICK_HEIGHT = 20` — height of tick marks in SVG units
- `const LABEL_FONT_SIZE = 14`
- `const ARC_COLOR = '#4ADE80'`

**NumberLineSvg.tsx (~150 lines):**
SVG rendering sub-component for tick marks, labels, and hop arrow arcs. Does NOT handle marker drag — that's in main component.

- Props: `{ range: [number, number]; width: number; lineY: number; hops: HopArrow[]; expandedDecade?: number | null; }`
- **Tick marks:** For each integer in range, render `<Line>` from react-native-svg. Major ticks (multiples of 5 or 10 depending on range) are taller. For 0-100 range, render ticks at multiples of 10 by default; when `expandedDecade` is set (e.g., 30), show individual ticks 30-40.
- **Labels:** `<SvgText>` below each tick. For 0-100, label every 10. For expanded decade, label each integer.
- **Hop arrows:** For each `HopArrow` in `hops`, render a quadratic Bezier arc (`<Path>`) above the number line. Use `M fromX lineY Q midX (lineY - arcHeight) toX lineY` pattern. Arc height scales with hop distance: `Math.min(30, Math.abs(toX - fromX) * 0.4)`. Each arc has a small arrowhead at the end (two short lines forming a V) and a "+1" (or "+N") label above the peak.
- **Base line:** Single horizontal `<Line>` across the full width at `lineY`.
- Helper: `valueToX(value: number, range: [number, number], width: number)` — maps a number value to pixel X coordinate. Linear mapping: `((value - range[0]) / (range[1] - range[0])) * width`.
- Export `valueToX` for use by main component.

**NumberLine.tsx (~300 lines):**
- Uses `ManipulativeShell` with `count` = current marker position value
- State:
  - `markerValue: number` — current tick position (integer), starts at `startPosition` or `range[0]`
  - `startValue: number` — where the marker started (for hop trail calculation)
  - `hops: HopArrow[]` — cumulative trail from start to current position
  - `expandedDecade: number | null` — for 0-100 range, which decade is expanded (null = overview)
- **Layout:** Use `onLayout` to measure the SVG container's width. The SVG fills the horizontal space with padding.
- **SVG container:** `<Svg width={containerWidth} height={SVG_HEIGHT}>` wrapping `<NumberLineSvg>`.
- **Marker:** Use a custom `Gesture.Pan()` (NOT DraggableItem, because the marker moves along a 1D axis constrained to the number line). The marker is an SVG `Circle` or a regular `Animated.View` positioned absolutely over the SVG.
  - Pan gesture: `onChange` computes nearest tick value from translationX + current pixel position. Use `xToValue(pixelX, range, width)` to convert. Clamp to range.
  - `onEnd`: Commit final position. Snap marker to nearest tick (discrete integer step). Update `markerValue` and rebuild `hops` array.
  - Snap animation: `withSpring` to the exact tick pixel position using `SNAP_SPRING_CONFIG`.
  - Fire `triggerSnapHaptic()` on each tick snap.
- **Hop trail:** When marker moves from `startValue` to `markerValue`, generate hop arrows for each unit step. For example, start=3, current=7 produces hops: [{from:3,to:4}, {from:4,to:5}, {from:5,to:6}, {from:6,to:7}]. This cumulative trail is the pedagogically important counting-on visualization.
- **0-100 range decade expansion:** When range is 0-100, show ticks at 10s. Tap on a decade segment (e.g., between 30 and 40) to expand it — set `expandedDecade = 30`. The SVG re-renders showing individual ticks 30-40. Marker snap still works within expanded section. Tap elsewhere to collapse back to overview.
- **SVG coordinate approach:** Skip `viewBox` — use pixel-based SVG coordinates matching the container's layout dimensions directly (avoids coordinate space mismatch pitfall from research). The SVG width equals the container width from onLayout.
- **Reset:** Set markerValue back to startValue or range[0], clear hops, collapse expanded decade.
- **Accessibility:** Marker has `accessibilityLabel="Number line marker at {markerValue}"`. Each hop shows value.
- Use `StyleSheet.create`.

**index.ts:** `export { NumberLine } from './NumberLine'`

- Color constants:
  - `FRACTION_PRIMARY = '#A855F7'` (purple per CONTEXT.md)
  - `FRACTION_SHADED = '#A855F7'` (solid purple for shaded sections)
  - `FRACTION_UNSHADED = 'rgba(168, 85, 247, 0.15)'` (faint purple for unshaded)
  - `FRACTION_BORDER = '#7C3AED'`
- `const MAX_STRIPS = 3` (up to 3 stacked per CONTEXT.md)
- `const MIN_SECTION_WIDTH = 48` (ensure 48dp touch targets)

**FractionStrips.tsx (~250 lines):**
- Uses `ManipulativeShell` with `count` = total shaded sections across all strips, `countLabel` = fraction display
- State:
  - `strips: StripState[]` — array of up to 3 strip states. Initially 1 strip with denominator 2, all unshaded.
- **Strip rendering:** Each strip renders as a horizontal row of equal-width `Pressable` sections. Stack strips vertically with `spacing.md` gap between them.
  - Section width: `stripWidth / strip.denominator` where `stripWidth` is measured from `onLayout`
  - Each section: `Pressable` with `onPress={() => toggleSection(stripIndex, sectionIndex)}`
  - Shaded sections: solid `FRACTION_SHADED` background
  - Unshaded sections: `FRACTION_UNSHADED` background
  - All sections: `borderWidth: 2`, `borderColor: FRACTION_BORDER`, `borderRadius: 4` on outer edges
  - Section height: `56dp` minimum for easy tapping
  - Strip total width fills available container width
- **Fraction label:** To the right of each strip, render `<Text>` showing `{numerator}/{denominator}` where `numerator = shaded.filter(Boolean).length`.
- **Tap to toggle:** `toggleSection(stripIdx, sectionIdx)` flips the boolean in `strips[stripIdx].shaded[sectionIdx]`. Fire `triggerSnapHaptic()` on toggle.
- **Adding strips:** "Add strip" button below the last strip (if `strips.length < MAX_STRIPS`). Opens a denominator selector — 5 buttons for each denominator (2, 3, 4, 6, 8). Selecting one adds a new strip with that denominator, all unshaded. Use a simple row of `Pressable` circles showing the number.
- **Removing strips:** Long-press on a strip's fraction label to remove it (or a small X button next to each strip). Minimum 1 strip.
- **Comparison:** Strips stacked vertically with equal total width — sections align proportionally for visual comparison (e.g., 1/2 visually aligns with 2/4).
- **ManipulativeShell count:** Total shaded sections across all strips. Label shows the fraction of the first strip: `"{shaded}/{denom}"`. If multiple strips, show first strip's fraction only (the visual comparison is the primary learning tool).
- **Reset:** Clear all strips, reset to 1 strip with denominator 2, all unshaded.
- **Touch target validation:** If `stripWidth / denominator < MIN_SECTION_WIDTH`, the strip should expand to be scrollable horizontally (wrap in `ScrollView horizontal`). But given the denominator cap of 8 and typical phone width (~360dp), `360 / 8 = 45dp` which is close to 48dp. Add `paddingHorizontal` to the container to account for this. If needed, reduce container padding to maximize strip width.
- **Accessibility:** Each section: `accessibilityLabel="{shaded ? 'Shaded' : 'Unshaded'} section {n} of {denominator}"`. Add strip button: `accessibilityLabel="Add fraction strip"`.
- Use `StyleSheet.create`. No inline styles.
