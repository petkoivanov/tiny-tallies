**BaseTenBlocks.tsx (~350 lines):**
- Uses `ManipulativeShell` with `count` = computed total value (hundreds*100 + tens*10 + ones)
- State:
  - `blocks: BlockState[]` — array of all blocks on the mat
  - `containerWidth: number` — measured from onLayout
  - `isGrouping: boolean` — true during auto-group animation (prevents adding more blocks)
  - `capMessage: string | null` — "Try grouping your cubes!" when at MAX_OBJECTS
- Refs:
  - `autoGroupTimer: useRef<ReturnType<typeof setTimeout> | null>(null)` — single timer for auto-group (pitfall 1)
  - `registeredOffsets: useRef<Map<string, { offsetX: SharedValue; offsetY: SharedValue }>>()` — for reset

**Place-Value Mat Layout:**
- 3 `SnapZone` columns side by side, each ~1/3 width:
  - Hundreds (left): `<SnapZone id="col-hundreds" ...>` with `COLUMN_COLORS.hundreds` background, "Hundreds" label at top
  - Tens (middle): `<SnapZone id="col-tens" ...>` with `COLUMN_COLORS.tens` background, "Tens" label
  - Ones (right): `<SnapZone id="col-ones" ...>` with `COLUMN_COLORS.ones` background, "Ones" label
- `snapTargets` SharedValue: array of 3 SnapTargets (one per column). Updated via `onMeasured` from each SnapZone.

**Block Tray (bottom):**
- Fixed-height area below the mat with 3 source blocks (one of each type): CubeBlock, RodBlock, FlatBlock.
- Each source is tappable (`Pressable`): tap adds a new block of that type to the appropriate column (cube -> ones, rod -> tens, flat -> hundreds).
- Also draggable from tray to mat (create a new `DraggableItem` on drag start from tray). Use a "source" DraggableItem that when snapped to a column, creates a new block in state and resets the source back to tray position.
- **30-object cap:** Before adding, check `blocks.length >= MAX_OBJECTS`. If at cap, set `capMessage`. Derive capMessage reactively from `blocks.length` (so it auto-clears when auto-group reduces count, avoiding pitfall 4 from research).

**Block Rendering:**
- Each block in `blocks` is rendered as a `DraggableItem` wrapping the appropriate renderer (`CubeBlock`, `RodBlock`, `FlatBlock`).
- Position blocks within their column using `getBlockPosition()` from layout helpers.
- `onRegister` callback stores offsetX/offsetY SharedValues in `registeredOffsets` map for reset support.

**onSnap Handler (critical):**
- When a block is snapped to a column, determine if it's a cross-column move:
  - Rod snapped to ones column -> **cross-column decompose**: remove the rod, add 10 cubes to ones column. Fire `triggerGroupHaptic()`.
  - Flat snapped to tens column -> **cross-column decompose**: remove the flat, add 10 rods to tens column.
  - Otherwise: update the block's `column` in state.
- After any snap, call `checkAutoGroup()`.

**Auto-Group Logic (MANIP-02):**
- `checkAutoGroup()`:
  1. Clear any existing timer: `if (autoGroupTimer.current) clearTimeout(autoGroupTimer.current)`
  2. Count cubes in ones column. If >= 10: `autoGroupTimer.current = setTimeout(performAutoGroupOnes, AUTO_GROUP_DELAY)`
  3. Count rods in tens column. If >= 10: `autoGroupTimer.current = setTimeout(performAutoGroupTens, AUTO_GROUP_DELAY)`
  4. Only one timer active at a time (ones takes priority).
- `performAutoGroupOnes()`:
  1. Set `isGrouping = true` to prevent concurrent modifications
  2. Animate merge: use `withDelay` and `withTiming` on the 10 cubes' SharedValue offsets to slide them toward a convergence point (staggered, 20ms per cube, 200ms slide duration). Then 200ms morph (scale down).
  3. After animation (~400ms total): Remove 10 cube BlockStates, add 1 rod BlockState to tens column. `triggerGroupHaptic()`.
  4. Set `isGrouping = false`. Call `checkAutoGroup()` again (in case remaining cubes still >= 10).
- `performAutoGroupTens()`: Same pattern — 10 rods merge into 1 flat in hundreds column.
- **Animation approach:** The merge animation modifies the SharedValue offsets of existing DraggableItems (which are registered via `onRegister`). After the visual animation completes, swap React state (remove 10, add 1). Use `setTimeout` for the post-animation state swap (animation duration is known: ~400ms).

**Tap-to-Decompose:**
- `onTap` handler on each block's DraggableItem:
  - Tap a rod -> `decomposeRod(blockId)`: remove the rod, add 10 cubes to ones column. Fan-out animation: cubes appear from rod's position and spread out. Use `FadeIn.delay(i * 30)` layout animation on each new cube for staggered appearance.
  - Tap a flat -> `decomposeFlat(blockId)`: remove the flat, add 10 rods to tens column. Same staggered appearance.
  - Tap a cube -> no-op (cubes can't decompose further).
  - Fire `triggerGroupHaptic()` on decompose.

**Reset:**
- Animate all blocks back to tray/start positions using registered SharedValue offsets + `withSpring(0, RESET_SPRING_CONFIG)` with stagger.
- After animation, clear `blocks` state.
- Clear any auto-group timer.

**Cap Message:**
- Derive from `blocks.length`: `capMessage = blocks.length >= MAX_OBJECTS ? "Try grouping your cubes!" : null`
- Render as a `<Text>` above the tray when non-null. Gentle, encouraging tone.

**Accessibility:**
- Each block: `accessibilityLabel="{type} block in {column} column"` (e.g., "cube block in ones column")
- Tray sources: `accessibilityLabel="Add {type}"` (e.g., "Add cube")
- Column labels visible and accessible

Use `StyleSheet.create` for all styles. Ensure file stays under 500 lines (target ~350). If approaching limit, extract auto-group logic into a separate `useAutoGroup` hook file.
