    **Gesture composition** (per locked decisions):
    - Use `Gesture.Race(tap, pan)` -- tap completes before pan activates
    - Tap gesture: `onEnd` calls `scheduleOnRN(onTap, id)` for tap-to-add/remove
    - Pan gesture: `minDistance(8)` to prevent accidental drags (ages 6-7 finger rests)
    - Pan.onStart: save current offset to startX/startY shared values, set isDragging=true, call `scheduleOnRN(onDragStart, id)` if provided
    - Pan.onChange: update offsetX/offsetY = startX/startY + translationX/Y (all on UI thread)
    - Pan.onEnd: set isDragging=false, call `findNearestSnap()` from snapMath worklet. If target found: animate to target center with `withSpring(SNAP_SPRING_CONFIG)`, call `scheduleOnRN(onSnap, id, target.id)` and `scheduleOnRN(triggerSnapHaptic)`. If no target: animate back to start with `withSpring(RETURN_SPRING_CONFIG)`.

    **Position approach** (per locked decision):
    - Use `transform: [{ translateX: offsetX.value }, { translateY: offsetY.value }]` -- NEVER `left`/`top`
    - Use `useSharedValue` for offsetX, offsetY, startX, startY, isDragging

    **Visual feedback during drag** (Claude's discretion):
    - Scale up to `DRAG_SCALE` (1.08) during drag via `withTiming` with 100ms duration
    - Reduce opacity to `DRAG_OPACITY` (0.85) during drag
    - Set zIndex to 999 during drag (so dragged item floats above others)

    **Animated style:**
    ```typescript
    const animatedStyle = useAnimatedStyle(() => ({
      transform: [
        { translateX: offsetX.value },
        { translateY: offsetY.value },
        { scale: withTiming(isDragging.value ? DRAG_SCALE : 1, { duration: 100 }) },
      ],
      opacity: withTiming(isDragging.value ? DRAG_OPACITY : 1, { duration: 100 }),
      zIndex: isDragging.value ? 999 : 0,
    }));
    ```

    **Reset support:**
    Export a `useResetDraggable` hook or expose reset via ref pattern. Since manipulatives own multiple DraggableItems, the simplest approach is: DraggableItem exposes its `offsetX` and `offsetY` shared values via a callback prop `onRegister?: (id: string, offsets: { offsetX: SharedValue<number>; offsetY: SharedValue<number> }) => void`. The parent collects these and can reset all by setting values to 0 (with spring or instant). This supports MANIP-09.

    **Thread crossing** (per locked decision):
    - Use `scheduleOnRN` from `react-native-worklets` -- NOT `runOnJS`
    - Arguments passed as separate params: `scheduleOnRN(fn, arg1, arg2)` -- NOT `scheduleOnRN(fn(arg))`
    - Only call scheduleOnRN in onEnd (not onChange/onUpdate per anti-pattern)

    **Enabled prop:** When `enabled=false`, gesture should be disabled. Use `.enabled(enabledValue)` on both tap and pan gestures. Bridge the `enabled` prop through a SharedValue so the worklet reads current value.

    **Accessibility:**
    - `accessible={true}`, `accessibilityRole="button"`
    - Use provided `accessibilityLabel`
    - Minimum touch target 48dp via `minWidth: 48, minHeight: 48` in default style

    **Styling:** Use `StyleSheet.create` for component styles (per CLAUDE.md). Dark theme-friendly colors.
