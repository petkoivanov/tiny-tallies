  <behavior>
    - useActionHistory: pushState adds current to history and sets new current
    - useActionHistory: undo pops last history item and sets as current, returns restored state
    - useActionHistory: undo returns null when history is empty
    - useActionHistory: canUndo is true when history has items, false when empty
    - useActionHistory: history is capped at maxSteps (default 10), oldest items dropped
    - useActionHistory: reset clears history and sets new initial state
    - guidedSteps: getNextGuidedStep returns correct target for ten_frame + addition
    - guidedSteps: getNextGuidedStep returns correct target for counters + addition
    - guidedSteps: getNextGuidedStep returns correct target for base_ten_blocks + addition
    - guidedSteps: getNextGuidedStep returns null when all steps complete
    - guidedSteps: getNextGuidedStep returns null for unmapped manipulative types
  </behavior>
  <action>
    1. Create `src/services/cpa/guidedStepsTypes.ts`:
       - `GuidedStep` interface: `{ targetId: string; hintText?: string }`
       - `GuidedStepResolver` interface: `{ operation: Operation; manipulativeType: ManipulativeType; getNextStep: (operands: [number, number], currentCount: number) => GuidedStep | null }`
       - Import Operation from mathEngine types, ManipulativeType from cpaTypes

    2. Create `src/services/cpa/guidedSteps.ts`:
       - Implement `getNextGuidedStep(operation, manipulativeType, operands, currentCount): GuidedStep | null`
       - Internal resolver registry: start with 5 most common pairings:
         - counters + addition: targetId = 'add-counter-button', count from 0 to operands[0]+operands[1]
         - counters + subtraction: targetId = 'add-counter-button' for first operand, then 'counter to flip' for removing
         - ten_frame + addition: targetId = `cell-${currentCount}` until sum reached
         - ten_frame + subtraction: targetId = cell to remove from filled cells
         - base_ten_blocks + addition: targetId = appropriate column button based on digit decomposition
         - number_line + addition: targetId = 'marker' with hint about target position
         - bar_model + addition: generic 'whole-label' target
       - Return null for unmapped combinations
       - Each resolver uses function-based approach `(operands, currentCount) => GuidedStep | null` per CONTEXT.md decision to handle state changes dynamically

    3. Create `src/components/manipulatives/shared/useActionHistory.ts`:
       - Generic hook `useActionHistory<T>(initialState: T, maxSteps = 10)`
       - Internal state: `{ history: T[], current: T }`
       - `pushState(newState: T)`: pushes current to history (capped at maxSteps via slice(-maxSteps)), sets newState as current
       - `undo(): T | null`: pops last from history, sets as current, returns it. Returns null if empty.
       - `reset(resetState: T)`: clears history, sets resetState as current
       - Returns `{ state, canUndo, historyLength, pushState, undo, reset }`
       - Use useState + useCallback pattern (not useReducer) for simplicity

    4. Create test file `src/__tests__/manipulatives/useActionHistory.test.ts`:
       - Use `renderHook` from @testing-library/react-native
       - Test all behaviors listed above
       - Ensure `act()` wraps state updates

    5. Create test file `src/__tests__/cpa/guidedSteps.test.ts`:
       - Test getNextGuidedStep for each mapped combination
       - Test null return for completed steps and unmapped types

    6. Update `src/services/cpa/index.ts`: add exports for `getNextGuidedStep`, `GuidedStep`, `GuidedStepResolver`
  </action>
  <verify>
    <automated>npm test -- --testPathPattern="useActionHistory|guidedSteps" --no-coverage</automated>
  </verify>
