    3. Replace every `setState(newValue)` call with `pushState(newValue)` -- these are the user actions that should be undoable

    6. Pass `onUndo={handleUndo}` and `canUndo={canUndo}` to `ManipulativeShell`

    1. Import `GuidedHighlight` from `../shared`
    2. Add optional `guidedTargetId?: string | null` to each component's props interface (in the corresponding Types file or inline)
    3. In the render logic, for each interactive element that has a semantic ID (SnapZone, cell, counter, button, marker), check if that element's ID matches `guidedTargetId`
    4. If it matches, wrap that element with `<GuidedHighlight active={true}>...</GuidedHighlight>`
    5. If it does not match, render normally (no GuidedHighlight wrapper, or `<GuidedHighlight active={false}>`)

    **Counters** (`CounterState[]`):

    - `handleFlip`: `pushState(counters.map(c => ...))`
    - `handleMove`: `pushState(counters.map(c => ...))`
    - `handleReset`: `reset([]); nextId.current = 0;`

    - `handleSnap`: `pushState(next)` where next is the new cells array
    - `handleCellTap`: `pushState(next)`
    - `handleReset`: `reset(new Array(CELLS_PER_FRAME).fill(false)); setFrameCount(1); ...`

       Also track a `guidedHintText` state: `const [guidedHintText, setGuidedHintText] = useState<string | null>(null)`

    3. Compute guided step when in concrete mode:
       ```
       useEffect(() => {
         if (stage === 'concrete' && manipulativeType) {
           const step = getNextGuidedStep(
             problem.operation,
             manipulativeType,
             problem.operands,
             0  // Initial count -- will need to be tracked
           );
           setGuidedTargetId(step?.targetId ?? null);
           setGuidedHintText(step?.hintText ?? null);
         } else {
           setGuidedTargetId(null);
           setGuidedHintText(null);
         }
       }, [stage, manipulativeType, problem, currentIndex]);
       ```

    4. **CRITICAL: Pass `guidedTargetId` down to the manipulative component.** In `renderManipulative()`, pass the prop to the rendered manipulative component so the glow appears on the actual target zone/item inside the manipulative:
       ```
       if (manipulativeType) {
         const ManipulativeComponent = MANIPULATIVE_COMPONENTS[manipulativeType];
         const targetId = stage === 'concrete' ? guidedTargetId : null;
         return <ManipulativeComponent
           key={currentIndex}
           guidedTargetId={targetId}
           testID="session-manipulative"
         />;
       }
       ```
       This requires updating the MANIPULATIVE_COMPONENTS type to accept `guidedTargetId?: string | null`. Update the component map type:
       ```
       const MANIPULATIVE_COMPONENTS: Record<ManipulativeType, React.ComponentType<{ testID?: string; guidedTargetId?: string | null }>> = { ... };
       ```

       {guidedHintText && stage === 'concrete' && (

           <Text style={styles.guidedHintText}>{guidedHintText}</Text>
         </View>
       )}

       Note: The glow itself is on the actual target zone/item inside the manipulative (via guidedTargetId prop). The hint text is a supplementary label, NOT wrapped in GuidedHighlight.
