    ## Frustration Guard (frustrationGuard.ts)
    - FRUSTRATION_THRESHOLD = 3 (exported constant)
    - createFrustrationState(): returns empty FrustrationState { consecutiveWrong: {} }
    - updateFrustrationState(state, skillId, correct): returns new FrustrationState
      - correct=true: removes skillId from consecutiveWrong (reset counter)
      - correct=false: increments consecutiveWrong[skillId] (or sets to 1 if absent)
    - shouldTriggerGuard(state, skillId): returns boolean
      - Returns true when consecutiveWrong[skillId] >= FRUSTRATION_THRESHOLD (3)
      - Returns false otherwise
    - All functions are pure -- no mutation of input state, return new objects
    - State is session-scoped and ephemeral (NOT persisted in store)

    - UNLOCK_THRESHOLD = 950 (exported constant)
    - isSkillUnlocked(skillId, skillStates, defaultElo?): returns boolean
      - Root skills (prerequisites.length === 0) are always unlocked
      - Returns true only when ALL prerequisite skills have eloRating >= UNLOCK_THRESHOLD
      - Uses defaultElo (default 1000) for skills not yet in skillStates
      - Returns false if skillId not found in SKILLS array
