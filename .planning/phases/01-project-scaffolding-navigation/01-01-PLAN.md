<task type="auto">
  <name>Task 2: Create Zustand store skeleton with four domain slices</name>
  <files>src/store/slices/childProfileSlice.ts, src/store/slices/skillStatesSlice.ts, src/store/slices/sessionStateSlice.ts, src/store/slices/gamificationSlice.ts, src/store/appStore.ts</files>
  <action>
Per user decisions: types co-located in each slice file, single appStore.ts composes all slices via spread, no persistence middleware, store version tracking.

Create each slice using Zustand 5's StateCreator pattern. Each slice file exports its interface type AND its creator function. Each slice uses `import type { AppState } from '../appStore'` for the StateCreator generic (type-only circular import is safe — types are erased at compile time).

**childProfileSlice.ts:**
- Interface: childName (string | null), childAge (number | null), childGrade (number | null), avatarId (string | null)
- Action: setChildProfile(profile: Partial<Pick<ChildProfileSlice, 'childName' | 'childAge' | 'childGrade' | 'avatarId'>>) — merges partial profile into state
- All fields init to null

**skillStatesSlice.ts:**
- Define a SkillState type: { eloRating: number; attempts: number; correct: number }
- Interface: skillStates (Record<string, SkillState> initialized to empty object {})
- Action: updateSkillState(skillId: string, update: Partial<SkillState>) — merges update into skillStates[skillId]
- Action: resetSkillStates() — resets to empty object

**sessionStateSlice.ts:**
- Interface: isSessionActive (boolean, init false), currentProblemIndex (number, init 0), sessionScore (number, init 0), sessionXpEarned (number, init 0), sessionAnswers (Array<{ problemId: string; answer: number; correct: boolean }>, init empty array)
- Action: startSession() — sets isSessionActive true, resets index/score/xp/answers
- Action: endSession() — sets isSessionActive false
- Action: recordAnswer(answer: { problemId: string; answer: number; correct: boolean }) — appends to sessionAnswers, increments score if correct, increments currentProblemIndex

**gamificationSlice.ts:**
- Interface: xp (number, init 0), level (number, init 1), weeklyStreak (number, init 0), lastSessionDate (string | null, init null)
- Action: addXp(amount: number) — adds to xp
- Action: setLevel(level: number) — sets level
- Action: incrementStreak() — increments weeklyStreak
- Action: resetStreak() — sets weeklyStreak to 0

**appStore.ts:**
- Define `AppState` as intersection of all four slice interfaces: ChildProfileSlice & SkillStatesSlice & SessionStateSlice & GamificationSlice
- Export `STORE_VERSION = 1` with comment: "Increment + add migration function when changing schema shape (see CLAUDE.md guardrail)"
- Create store: `export const useAppStore = create<AppState>()((...a) => ({ ...createChildProfileSlice(...a), ...createSkillStatesSlice(...a), ...createSessionStateSlice(...a), ...createGamificationSlice(...a) }))`
- Import create from 'zustand', import each slice creator and type

Do NOT add persist middleware — that is Phase 4 per user decision.
