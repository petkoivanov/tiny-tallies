**src/services/mathEngine/generator.ts:**
This is the core of the engine. Implements `generateProblem()` and `generateProblems()`.

Import:
- `z` from 'zod' (for Zod v4 schema validation at API boundary)
- `createRng` from './seededRng'
- `requiresCarry as checkCarry, requiresBorrow as checkBorrow` from './constraints' (renamed to avoid collision with template fields)
- `findTemplate, getTemplatesBySkill` from './templates'
- Types: `Problem, ProblemTemplate, GenerationParams, BatchGenerationParams` from './types'

Define Zod schemas (Zod v4 syntax — no deprecated patterns):
```typescript
export const GenerationParamsSchema = z.object({
  templateId: z.string(),
  seed: z.number().int(),
});

export const BatchGenerationParamsSchema = z.object({
  skillId: z.string(),
  count: z.number().int().min(1).max(50),
  seed: z.number().int(),
});
```

Define MAX_ATTEMPTS constant = 100 (module-level, not exported).

Implement internal helper `generateOperands(template: ProblemTemplate, rng: SeededRng): [number, number]`:
- Loop up to MAX_ATTEMPTS times.
- On each iteration:
  1. Generate `a = rng.intRange(template.operandRanges[0].min, template.operandRanges[0].max)`
  2. For subtraction: constrain `b` upper bound to `Math.min(template.operandRanges[1].max, a - 1)` to ensure non-negative result. If this constrained max < `template.operandRanges[1].min`, continue to next attempt.
  3. Generate `b = rng.intRange(template.operandRanges[1].min, bMax)` where `bMax` is the (possibly constrained) max.
  4. Check carry constraint: if `template.requiresCarry === true` and `!checkCarry(a, b)`, continue. If `template.requiresCarry === false` and `checkCarry(a, b)`, continue.
  5. Check borrow constraint: if `template.requiresBorrow === true` and `!checkBorrow(a, b)`, continue. If `template.requiresBorrow === false` and `checkBorrow(a, b)`, continue.
  6. Compute result: `template.operation === 'addition' ? a + b : a - b`
  7. Check result range: if result < template.resultRange.min or result > template.resultRange.max, continue.
  8. Return `[a, b]`.
- If loop exhausts without finding valid operands, throw: `Error(\`Failed to generate operands for template ${template.id} after ${MAX_ATTEMPTS} attempts. Check that operand ranges are compatible with carry/borrow constraints.\`)`

Implement internal helper `formatQuestion(operation: Operation, a: number, b: number): string`:
- If addition: return `${a} + ${b} = ?`
- If subtraction: return `${a} - ${b} = ?`

Implement internal helper `computeAnswer(operation: Operation, a: number, b: number): number`:
- If addition: return a + b
- If subtraction: return a - b

Export `generateProblem(params: GenerationParams): Problem`:
1. Validate params with `GenerationParamsSchema.parse(params)` — Zod throws on invalid input.
2. `const template = findTemplate(params.templateId)`
3. `const rng = createRng(params.seed)`
4. `const [a, b] = generateOperands(template, rng)`
5. `const correctAnswer = computeAnswer(template.operation, a, b)`
6. Return Problem object:
   ```
   {
     id: `${template.id}_${params.seed}`,
     templateId: template.id,
     operation: template.operation,
     operands: [a, b],
     correctAnswer,
     questionText: formatQuestion(template.operation, a, b),
     skillId: template.skillId,
     standards: template.standards,
     grade: template.grades[0],
     baseElo: template.baseElo,
     metadata: {
       digitCount: template.digitCount,
       requiresCarry: template.requiresCarry ?? false,
       requiresBorrow: template.requiresBorrow ?? false,
     },
   }
   ```

Export `generateProblems(params: BatchGenerationParams): Problem[]`:
1. Validate params with `BatchGenerationParamsSchema.parse(params)`
2. Get templates for the skill: `const templates = getTemplatesBySkill(params.skillId)`
3. If templates is empty, throw: `Error(\`No templates found for skill ${params.skillId}\`)`
4. Generate `params.count` problems. For each index i:
   - Pick template: `templates[i % templates.length]` (round-robin if multiple templates per skill)
   - Derive seed: `params.seed + i` (each problem gets a unique seed)
   - Call `generateProblem({ templateId: template.id, seed: derivedSeed })`
5. Return array of Problem objects.
