<objective>
Enrich AI tutor prompts with confirmed misconception data so the LLM gives targeted explanations addressing specific misunderstanding patterns rather than generic help.

Purpose: Children with confirmed misconceptions get tutoring that directly addresses their specific struggles (e.g., forgetting to carry, subtracting smaller from larger) instead of generic hints. This is the key value of the misconception detection pipeline built in phases 26-28.

Output: Extended prompt types, misconception-aware prompt builders with per-mode guidance, and store-to-prompt data threading in useTutor.
</objective>
