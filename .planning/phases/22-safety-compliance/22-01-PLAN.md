    - checkAnswerLeak("think about what comes after 6", 7) returns { safe: true, reason: null } (6 is not the answer)
    - checkAnswerLeak("try adding 3 and 4", 7) returns { safe: true, reason: null } (operands, not answer)
    - checkAnswerLeak("the answer is 17", 7) returns { safe: true, reason: null } (17 != 7, word boundary)
    - checkAnswerLeak("you got seventy", 7) returns { safe: true, reason: null } (seventy != seven)
    - validateContent("Short sentence.", '6-7') returns { valid: true }
    - validateContent with 5 sentences returns { valid: false, reason: 'too_many_sentences' }
    - validateContent with a 12-word sentence for '6-7' bracket returns { valid: false, reason: 'sentence_too_long' }
    - validateContent with "understanding" (13 chars) for '6-7' returns { valid: false, reason: 'vocabulary_too_complex' }
    - scrubOutboundPii removes child name "Emma" from prompt, replacing with "the child"
    - scrubOutboundPii removes "7 years old" pattern, replacing with "the child"
    - scrubOutboundPii with null name and null age returns unchanged strings with piiFound=false
    - runSafetyPipeline returns { passed: true, text } when all checks pass
    - runSafetyPipeline returns { passed: false, fallbackCategory: 'answer_leaked' } when answer leaks
    - runSafetyPipeline returns { passed: false, fallbackCategory: 'content_invalid' } when content validation fails
  </behavior>
  <action>
    Create `src/services/tutor/safetyFilter.ts` with these pure functions:

    1. `checkAnswerLeak(response: string, correctAnswer: number): SafetyCheckResult`
       - Lowercase the response for case-insensitive matching
       - Pattern 1: Check for answer digit as standalone word boundary `\b${answerStr}\b`
       - Pattern 2: Check for spelled-out number word using `numberToWord(correctAnswer)`
       - Pattern 3: Check indirect phrasing patterns: "equals X", "is X", "get X", "makes X", "gives X" for both digit and word form
       - Return `{ safe: true, reason: null }` if no patterns match
       - IMPORTANT: Word boundary `\b` prevents matching "17" when answer is "7" or "seventy" when answer is "seven"

    2. `validateContent(response: string, ageBracket: AgeBracket): ContentValidationResult`
       - Split response on sentence-ending punctuation `[.!?]+`
       - Check sentence count against MAX_SENTENCES (4)
       - Check word count per sentence against CONTENT_WORD_LIMITS[ageBracket]
       - Check word character length (letters only, strip punctuation) against MAX_WORD_LENGTH[ageBracket]
       - Return first failure found, or `{ valid: true, reason: null }`

    3. `scrubOutboundPii(systemInstruction: string, userMessage: string, childName: string | null, childAge: number | null): { systemInstruction: string; userMessage: string; piiFound: boolean }`
       - If childName is non-null and non-empty, replace occurrences (case-insensitive) with "the child"
       - If childAge is non-null, replace patterns like "7 years old", "age: 7", "7-year" with "the child"
       - Return cleaned strings + piiFound flag

    4. `runSafetyPipeline(response: string, correctAnswer: number, ageBracket: AgeBracket): SafetyPipelineResult`
       - Orchestrates checkAnswerLeak then validateContent
       - If answer leaks: return `{ passed: false, fallbackCategory: 'answer_leaked', reason: checkResult.reason }`
       - If content invalid: return `{ passed: false, fallbackCategory: 'content_invalid', reason: contentResult.reason }`
       - If all pass: return `{ passed: true, text: response }`

    Write comprehensive tests in `src/services/tutor/__tests__/safetyFilter.test.ts`:
    - checkAnswerLeak: digit match, word match, indirect patterns, false positives (operands, word boundaries), large numbers (100+)
    - validateContent: happy path, too many sentences, sentence too long per bracket, complex vocabulary per bracket
    - scrubOutboundPii: name removal, age pattern removal, null inputs, no-PII inputs
    - runSafetyPipeline: full pass, answer leak, content invalid
  </action>
