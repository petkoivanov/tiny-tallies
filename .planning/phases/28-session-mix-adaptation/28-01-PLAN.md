<objective>
Modify the session practice mix algorithm to inject remediation problems for skills with confirmed misconceptions, replacing review slots (up to 3). Thread misconception data from the store through useSession -> generateSessionQueue -> generatePracticeMix.

Purpose: Children with confirmed misconceptions get targeted practice on those specific skills every session, accelerating misconception correction without disrupting forward progress (new/challenge slots preserved).

Output: Updated practice mix pipeline with 'remediation' category, backward-compatible when no misconceptions exist, fully tested.
</objective>

   - Use `adjustedReviewCount` instead of `slots.review` in the review fill loop
