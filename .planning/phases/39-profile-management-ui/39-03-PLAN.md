    1. Create `src/screens/ProfileManagementScreen.tsx`:
       - Entire screen wrapped in `<PinGate onCancel={() => navigation.goBack()} title="Manage Profiles">`
       - Internal view modes: 'list' | 'editing' | 'deleting'

       **List mode:**
       - Header: "Manage Profiles" with X close button (calls navigation.goBack)
       - ScrollView of profile cards, each showing:
         - AvatarCircle (small), child name, "Age {age}, Grade {grade === 0 ? 'K' : grade}"
         - Edit button (Edit2 icon from lucide) -> sets editing mode with that child's data
         - Delete button (Trash2 icon from lucide) -> sets deleting mode with that child

       - Profile count: "X of 5 profiles"

       **Editing mode:**
       - Render `<ProfileCreationWizard initialValues={editChild} onComplete={handleEditComplete} onCancel={() => setMode('list')} />`
       - `handleEditComplete`: calls `updateChild(editingChildId, { childName, childAge, childGrade, avatarId })`, then sets mode back to 'list'

       **Deleting mode:**
       - Modal overlay with:
         - Warning icon and "Delete {childName}'s profile?"
         - "This will permanently delete all learning progress, achievements, and settings."
         - TextInput: "Type {childName} to confirm"
         - Delete button: enabled when `input.toLowerCase() === childName.toLowerCase()`
         - Cancel button: returns to list mode
       - On delete confirm:
         - Call `removeChild(childId)`
         - Check remaining children count BEFORE removal (since removeChild is synchronous and we read from the closure):
           `if (Object.keys(children).length <= 1)` -> `navigation.reset({ index: 0, routes: [{ name: 'ProfileSetup' }] })`
         - Otherwise return to list mode

       Use StyleSheet.create for all styles. Use lucide-react-native for icons (Edit2, Trash2, UserPlus, X, AlertTriangle).
