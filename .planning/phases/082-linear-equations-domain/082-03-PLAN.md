From src/services/mathEngine/wordProblems/generator.ts (prefix mode behavior):
- prefix mode: returns { text: contextSentence + ' ' + originalQuestionText, question: originalQuestion }
- {name}, {name2}, {place}, {object}, {verb} are substitution vars from the word problem system
- {a} and {b} are NOT used in prefix mode (they reference operands — see Research Pitfall 5)
- Use {name} for student name, {place} for location context

RESEARCH NOTE on prefix mode: mode='prefix' prepends the template string before the original equation text (e.g., "Maya is figuring out an age puzzle. x + 5 = 12. Solve for x."). The equation itself comes from the generator's questionText — the word problem template just adds a scene-setting sentence. This sidesteps the operand display problem (Pitfall 5 in RESEARCH.md).
