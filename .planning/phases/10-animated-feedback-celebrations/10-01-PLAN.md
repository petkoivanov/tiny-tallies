<task type="auto">
  <name>Task 1: Create AnswerFeedbackAnimation component</name>
  <files>src/components/animations/AnswerFeedbackAnimation.tsx</files>
  <action>
Create a reanimated wrapper component that animates its children based on feedback type.

Props interface:
- `feedbackType: 'correct' | 'incorrect' | null` — drives which animation plays
- `children: React.ReactNode` — the answer button content

Correct animation (bounce + scale pulse):
- Use `useSharedValue` for scale, starting at 1.0
- On feedbackType='correct': `withSequence(withSpring(1.1, { damping: 4, stiffness: 300 }), withSpring(1.0, { damping: 8 }))`
- This gives a satisfying springy bounce: 1.0 -> 1.1 -> 1.0
- Apply via `useAnimatedStyle` with `transform: [{ scale }]`

Incorrect animation (gentle shake):
- Use `useSharedValue` for translateX, starting at 0
- On feedbackType='incorrect': `withSequence(withTiming(-6, { duration: 50 }), withTiming(6, { duration: 50 }), withTiming(-4, { duration: 50 }), withTiming(4, { duration: 50 }), withTiming(0, { duration: 50 }))`
- This gives 2 oscillations that decay: -6, +6, -4, +4, 0 (total ~250ms)
- Apply via `useAnimatedStyle` with `transform: [{ translateX }]`

Use `useEffect` watching `feedbackType` to trigger the appropriate animation. When feedbackType becomes null, reset shared values to defaults (scale=1, translateX=0).

Wrap children in `Animated.View` from react-native-reanimated (NOT from react-native).

Use StyleSheet.create for any static styles. Keep component under 80 lines.
  </action>

    npm run typecheck
  </verify>
  <done>AnswerFeedbackAnimation component exports correctly, accepts feedbackType prop, applies bounce on correct and shake on incorrect using reanimated shared values.</done>
